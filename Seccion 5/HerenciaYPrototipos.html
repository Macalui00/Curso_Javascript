Section 5: Herencia y Cadena de Prototipos

Herencia y Cadena de Prototipos

En esta leccion vamos a ver de cerca a los objetos en Javascript.
Vamos a hablar de POO y luego especificamente en JavaScript.

Hay un dicho en JavaScript que dice que "Todo es un Objeto"
Esto es casi todo correcto porque en JavaScript hay dos tipos de valores:
- Primitivos
- Objetos

Primitivos: son todos los valores que tienen alguno de los tipos de datos qe ya hemos mencionado anteriormente:
- Numeros
- Strings o Cadenas
- Booleanos
- Indefinido o Undefined
- Null

Y, todo lo demas, son objetos:
- Arrays o Listas
- Funciones
- Objetos
- Dates o Fechas
- Wrappers para Numbers, Strings, Booleans

Es por eso que en JavaScript se dice que todo es un objeto, y de hecho, incluso cosas muy simples como lo son
funciones, arrays o dates son todos objetos en javascript. Y esto es porque en Javascript, objetos no son solo para
almacenar un monton de informacion sino tambien para cosas realmente complejas bajo la superficie.

Entonces, estuvimos usando objetos todo el tiempo, sin saberlo.
Esto es una de las cosas que hace a JavaScript tan unico y diferente a otros lenguajes de programación populares.

POO:
- hace un uso intensivo de objetos, propiedades y metodos y estos objetos interactuan unos con otros para hacer
aplicaciones complejas
- usamos objetos para almacenar datos, estructurar nuestro código y mantener nuestro código limpio.

Hasta ahora, solo hemos creado objetos simples para almacenar cosas.
Y los hemos estado escribieno uno por uno. PERO HAY UNA MEJOR MANERA, imaginemos algo como un modelo desde el cual podamos generar tantos objetos como deseemos.
Y, podemos hacer eso en JavaScript.

Acá hay un ejemplo de esto:
Person {
name,
yearOfBirth,
job,
calculateAge()
}

Esto es un objeto especial persona que basicamente podemos usar como un modelo para crear muchos objeto persona.
En otros lenguajes de programación esto se denomina muchas veces CLASE, pero en JavaScript, nos gusta llamarlo CONSTRUCTOR o PROTOTIPO.
Por lo tanto, basandonos en este constructor podemos crear tantas instancias como queramos.

Herencia: es cunado un objeto se basa en otro objeto. Cuando un objeto tiene acceso a las propiedades y metodos de otro objeto.
Entonces imaginemos que ademas del constructor persona, queramos tener un constructor atleta con un par de propiedades y metodos diferentes.
Atleta{
olimpiadas,
medallasOlimpiadas,
olimpiadasPermitidas()
}
El atleta es tambien es una persona, por lo cual podemos usar herencia, hacemos que herede las propiedades y metodos del objeto persona.

Ahora veamos en particular, como JavaScript maneja la herencia con nuestros dos ejemplos, persona y Atleta
JavaScript es un lenguaje basado en PROTOTIPOS, que significa que la herencia funciona mediante el uso de algo llamado prototipos.
En la práctica, significa que todos y cada uno de los objetos JavaScript tienen una propiedad Prototipo que hace posible la herencia en
JavaScript.

Veamos mejor como funciona realmente la herencia...
Entonces, Persona es nuestro constructor y John es nuestra instancia, si queremos que john herede un metodo o una propiedad del objeto Persona,
tenemos que agregar ese metodo o propiedad a la propiedad prototipo persona, tal y como esta hecho en la imagen donde tengo un cuadradro persona
del cual deriva un prototipo en linea punteada, el cual incluye el metodo calcularEdad(). Por esto mismo, john hereda el metodo y puede llamarlo.
Y cualquier otro objeto creado por el constructor Persona, heredara tambien este metodo.
Lo que es importante resaltar acá es que el prototipo de Persona no es el prototipo de la Persona en si mismo, sino de todas las instancias que
cree a traves del plano Persona, como en este caso, John. Entonces, en otras palabras, la propiedad prototipo de la persona es el prototipo de
john. Y eso no es todo, porque el objeto Persona en sí es la instancia de un constructor aun mas grande que es el objeto Object:

Object tiene una propiedad prototipo que contiene:
hasOwnProperty(),
isPrototypeOf(),
...,
valueOf()

Suena un poco loco pero es asi como funciona. Todos y cada uno de los objetos que creamos es una instancia del constructor de Objetos que tiene
un monton de métodos en su propiedad de prototipo. Y, como puede adivinar, el objeto Persona hereda estos metodos y puede llamarlos. Además, el
Objeto John hereda estos métodos y puede llamarlos. Y con esto, hemos explicado la cadena de prototipos.

Entonces, la cadena de prototipos es lo que hace posible toda esta herencia. Y aquí esta como funciona: cuando uno intenta acceder a un determinado
metodo o propiedad de un objeto, JavaScript primero intentara buscar el metodo en el mismo objeto mismo, si no lo encuentra ahi, entonces buscara
en el prototipo del objeto, que es la propiedad prototipo de su elemento primario. Por lo tanto, se mueve hacia arriba en la cadena de prototipos.
Si el prototipo no esta ahí tampoco, esto continua hasta que no haya más prototipos para mirar, que es nulo. Nulo es el unico que no tiene prototipo
y, por lo tanto, es el último eslabón de la cadena de prototipos. Y en este caso, se devuelve indefinido.

Esta es la razon por la cual, por ejemplo, el objeto John podría llamar al método hasOwnProperty que está almacenado en la propiedad Object Prototype.
Y como esto es tan importante, aquí va un resumen:

- Cada objeto JavaScript tiene una propiedad prototipo, que hace posible la herencia en Javascript.
- La propiedad prototipo de un objeto es donde nosotros ponemos metodos y propiedades que queremos que otros objetos hereden.
- La propiedad Prototipo del Constructor no es el Prototipo del Constructor en sí, sino que es le prototipo de todas las instancias que se crean a
través de él.
- Cuando se llama a un determinado método, o propiedad, la busqueda comienza en el objeto mismo, y sino se puede encontrar ahí, la busqueda avanza
hacia el prototipo del objeto. Esto continúa hasta que el metodo es encontrado, y esta es la cadena de prototipos.
