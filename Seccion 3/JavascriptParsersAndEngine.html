Section 3: Como nuestro código es ejecutado:

Javascript Parsers and Engine

Esta es la primera lección teórica donde veremos muy rápidamente cómo se ejecutan realmente los códigos en Javascript que escribimos. Javascript siempre se aloja en algun entorno, ejemplos de estos entornos son: - Google Chrome - Firefox - Safari - etc. Allí es donde se ejecuta Javascript. Tambien pueden existir otros Hosts como pueden ser: - El servidor web NoJs O, incluso, algunas aplicaciones que aceptan la entrada de código Javascript. Sin embargo, siempre nos centraremos solo en los navegadores de este curso. Entonces, cuando escribimos nuestro código Javascript y realmente queremos ejecutarlo, hay muchas cosas sucediendo detrás de escena. Lo que sucede es que el host donde se aloja Javascript, tien algun tipo de motor de javascript que toma nuestro códio y lo ejecuta. Entonces, en terminos muy simples un motor de Javascript es un programa que ejecuta código Javascript. Hay muchos motores diferentes, como el motor V8 de Google, que se usa en Google Chrome. Pero, hay muchos otros. Como sabemos SpiderMonkey, nucleo de Javascript y muchos más. Ahora, lo primero que sucede dentro del motor, es que nuestro código es analizado por un analizador sintáctico, que, basicamente lee nuestro código linea por linea y verifica si la sintaxis del codigo que le dimos es correcta. Esto significa, que el analizador conoce las reglas de Javascript y como debe escribirse para que sea correcto y valido. Y si cometes algunos errores, basicamente arroja un error y detiene la ejecución. Sin embargo, si todo es correcto, el analizador genera una estructura de datos conocida como Árbol de sintaxis abstracta, que luego se traduce en código máquina. Entonces, este código ya no es código Javascript, sino un codigo, o digamos un conjunto de instrucciones, que puede ser ejecutado directamente por el procesador de la computadora. Y, es solo cuando nuestro código ya se convirtió en un código de máquina que, realmente se ejecuta y hace su trabajo. Entonces, no es realmente importante que realmente memorices todo esto. Solo que entiendas la idea basica de loq ue realmente sucede una vez que elegimos ejecutar nuestro código. Y, de hecho, hay mucho mas detras de escena. Y diferentes motores hacen las cosas ligeramente diferente. Pero, esto es absolutamente suficiente para entender el punto.
